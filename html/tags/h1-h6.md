# `<h1>` - `<h6>`

## Definition and Purpose

The `<h1>` through `<h6>` HTML elements define section headings in a document. They represent six levels of hierarchical headings: `<h1>` being the highest (most important) level and `<h6>` the lowest (least important).

These heading tags exist to impose semantic structure on a document; to indicate the organization of content (e.g., title, sections, subsections, sub-subsections) rather than to control styling (such as font size).

Proper use of headings supports readability, accessibility (e.g. screen readers can navigate via headings), and can aid search engines or other tools in understanding the document’s structure.

---

## Syntax and Variants

```html
<h1> … </h1>
<h2> … </h2>
<h3> … </h3>
<h4> … </h4>
<h5> … </h5>
<h6> … </h6>
```

**Content model**:

* The content of heading elements is restricted to **phrasing content** (i.e., inline content, such as text, inline markup, inline images, etc.).
* You may embed inline elements (such as `<em>`, `<strong>`, `<span>`), but **not** block-level elements (e.g., no `<div>`, `<p>` inside a heading).

---

## Attributes

* These heading tags do **not** define any heading-specific attributes. Only **global attributes** apply (e.g., `id`, `class`, `style`, `data-*`, `aria-*`, etc.)
* There are **no required attributes**.
* **Optional attributes**: any global attribute as needed by your application (e.g., `id` to anchor the heading, `class` for styling, `aria-*` for accessibility, etc.)
* **Deprecated/obsolete attributes**: The older `align` attribute (used historically for text alignment) is obsolete and should not be used; use CSS for styling instead.

---

## Examples

### Basic usage; full hierarchy

```html
<h1>Main Title of Document</h1>
<h2>Section 1</h2>
<h3>Subsection 1.1</h3>
<h4>Sub-subsection 1.1.1</h4>
<h5>Detail under sub-subsection</h5>
<h6>Lowest-level heading / fine detail</h6>
```

### Structuring a simple document

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>Sample Document</title>
</head>
<body>
  <h1>My Article: The Evolution of Web Standards</h1>
  
  <h2>Introduction</h2>
  <p>This article explores how web standards have evolved over the years...</p>
  
  <h2>Historical Background</h2>
    <h3>Early HTML (1990s)</h3>
    <p>HTML started as a simple markup system...</p>
    <h3>Growth and Diversity (2000–2010)</h3>
    <p>Multiple versions and competing browsers led to fragmentation...</p>
  
  <h2>Modern HTML and Best Practices</h2>
  <p>HTML5 introduced semantic elements, better accessibility, and structural clarity.</p>
</body>
</html>
```

### Heading with inline formatting

```html
<h2>Understanding <em>Semantic</em> HTML</h2>
<p>Semantic HTML helps machines and accessibility tools understand your content better.</p>
```
